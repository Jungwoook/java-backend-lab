# Repository Instructions

이 저장소는 Java와 Spring 백엔드 학습 내용을 Markdown 문서와 예제 코드로 정리하는 스터디 저장소다.

## 문서 작성 기준

개념 문서는 기본적으로 아래 템플릿을 따른다.

```markdown
# 제목

## 왜 배우는가

## 개념

## 핵심

## 코드 비교

## 동작 흐름

## 주의할 점

## 정리

## 다음으로 연결되는 주제
```

## 섹션 작성 원칙

- `왜 배우는가`: 이 개념이 해결하는 문제와 이후 학습 주제와의 연결점을 설명한다.
- `개념`: 정의, 배경, 주요 용어를 정리한다.
- `핵심`: 반드시 기억해야 할 판단 기준을 bullet로 정리한다.
- `코드 비교`: 필요한 경우에만 작성한다. 나쁜 예 / 개선한 예, 변경 전 / 변경 후, 절차적 코드 / 객체지향 코드처럼 비교한다.
- `동작 흐름`: 코드나 프레임워크가 어떤 순서로 실행되는지 설명이 필요할 때 작성한다.
- `주의할 점`: 흔한 실수, 오해, 남용하기 쉬운 부분을 정리한다.
- `정리`: 문서의 핵심을 짧게 복습한다.
- `다음으로 연결되는 주제`: 다음 학습 문서와 자연스럽게 이어지는 주제를 적는다.

`코드 비교`와 `동작 흐름`은 모든 문서에 억지로 넣지 않는다. 개념 이해에 도움이 될 때만 포함한다.

## 예제 코드 기준

- 문서 안에는 핵심을 이해할 수 있는 짧은 코드만 넣는다.
- 실행 가능한 예제나 코드가 길어지는 경우 `examples` 디렉터리에 별도 파일로 추가한다.
- 새 예제는 가능하면 아래 구조에 맞춘다.

```text
examples/
  java/
  oop/
  di/
  spring-core/
  spring-boot/
  mvc/
  jpa/
```

## 문체

- 설명은 학습자가 읽고 바로 이해할 수 있게 쉽게 쓴다.
- 단순 정의에서 끝내지 않고, 왜 그렇게 설계하는지까지 설명한다.
- 객체지향 설계 이후 주제는 Java 코드와 Spring 학습으로 이어질 수 있게 작성한다.
